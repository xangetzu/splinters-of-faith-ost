# AI Project Instructions

This project is a static website for hosting an original soundtrack.

When making changes:

-   Prioritize readability over cleverness.
-   Keep HTML semantic and accessible.
-   Prefer vanilla HTML, CSS, and JavaScript.
-   Avoid introducing frameworks unless explicitly requested.
-   Build reusable components where appropriate.
-   Keep styling organized and consistent.
-   Keep JavaScript modular.
-   Design for long-term maintainability.
-   Assume the soundtrack will continue growing over many years.

## Development Principles

-   Separate content from presentation.
-   Keep track data in structured metadata whenever practical.
-   Avoid hardcoding soundtrack entries into HTML.
-   Prefer scalable solutions over quick fixes.
-   Comment code where the intent may not be obvious.
-   Preserve backward compatibility whenever reasonable.

## User Experience Goals

The site should feel like an official soundtrack companion rather than a
generic music hosting service.

Prioritize:

-   Atmosphere
-   Simplicity
-   Discoverability
-   Fast loading
-   Elegant presentation
