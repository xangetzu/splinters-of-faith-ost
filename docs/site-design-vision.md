# Splinters of Faith Website Design Vision

Version: 0.1

## Purpose

This document describes the intended **experience** of the website
rather than its implementation.

The site should not feel like a generic music streaming service or a
file repository. It should feel like the official soundtrack companion
for a fantasy RPG.

------------------------------------------------------------------------

# Core Philosophy

Visitors should feel as though they are exploring the world of
**Splinters of Faith** through its music.

The music is the guide.

The world is the destination.

Think:

-   Game codex
-   World atlas
-   Fantasy companion book

Not:

-   Spotify
-   SoundCloud
-   A folder of MP3s

------------------------------------------------------------------------

# Overall Experience

Large artwork, elegant typography, subtle animation, and cinematic
presentation are encouraged.

The design should feel premium while remaining intuitive.

------------------------------------------------------------------------

# Navigation

Organize the soundtrack into major collections rather than albums.

Examples:

-   Main Suite
-   Biomes
-   Characters
-   Tavern Music
-   Stingers
-   Special Collections

Selecting a collection should feel like entering a new region of the
world.

------------------------------------------------------------------------

# Biomes

Each biome should have its own identity.

Examples:

-   Forest
-   Swamp
-   Mountains
-   Desert
-   Coast
-   Plains

Each biome should have:

-   Dedicated banner artwork
-   Subtle color palette
-   Atmospheric presentation
-   Track listing
-   Description
-   Play All button

The visitor should instantly recognize where they are based on the
artwork.

------------------------------------------------------------------------

# Characters

Character pages may include:

-   Portrait artwork
-   Theme music
-   Character description
-   Composer notes

------------------------------------------------------------------------

# Artwork

Artwork is a primary design element.

Every major section should have unique artwork.

Examples:

-   Home
-   Forest
-   Swamp
-   Mountains
-   Characters
-   Tavern Music
-   Main Suite

The artwork should define the atmosphere of the page.

------------------------------------------------------------------------

# Audio Player

The player should remain persistent throughout the website.

Changing pages should not interrupt playback.

Desired playback modes:

-   Play Entire Soundtrack
-   Narrative Order
-   World Order
-   Shuffle

------------------------------------------------------------------------

# Visual Style

Overall feeling:

-   Fantasy
-   Elegant
-   Atmospheric
-   Warm
-   Inviting

Glass-inspired UI elements are encouraged, but they should feel like
fantasy glass rather than a modern mobile operating system.

Artwork should always be the visual focus.

------------------------------------------------------------------------

# Home Page

The home page should feature large clickable artwork panels representing
the major soundtrack collections.

Examples:

-   Main Suite
-   Tavern Music
-   Stingers
-   Characters
-   Forest
-   Mountains
-   Swamp

Each panel serves as a portal into that section.

------------------------------------------------------------------------

# Technical Direction

Favor reusable components.

Keep metadata separate from presentation.

Avoid hardcoding track listings whenever possible.

The long-term goal is that adding new tracks should primarily involve
updating metadata rather than editing HTML.

------------------------------------------------------------------------

# Design Goal

The finished website should feel less like a music download site and
more like opening the official soundtrack companion to a fantasy
role-playing game.

Visitors should be encouraged to explore the world through its music.
